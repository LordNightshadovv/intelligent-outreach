# Outreach Policy

Use this reference before planning outreach in real communities, especially for politics, religion, health, identity, sexuality, law, or other sensitive subjects.

## Allowed Work

- Research public communities and discussions.
- Summarize platform or community rules visible to a normal visitor.
- Identify high-fit threads where the user's content can add genuine value.
- Draft context-aware replies for human review.
- Prepare an in-Codex approval pack.
- Log in to a user-authorized account with secure local credentials when platform rules permit it.
- Post an exact approved reply after the user approves the target, account, and final message in Codex chat.
- Recommend that the user create an account manually or disclose affiliation when necessary.

## Disallowed Work

- Automatically create accounts, vote, follow, DM, or send private messages.
- Log in or post without explicit user authorization for that platform/account and exact approval for that target/message.
- Use fake identities, sockpuppets, astroturfing, undisclosed affiliation, or simulated grassroots support.
- Scrape private groups, logged-in-only areas, DMs, paid spaces, or content behind access controls unless the user provides lawful exports and the platform permits the intended use.
- Evade bans, captchas, rate limits, moderation queues, spam filters, bot detection, or robots restrictions.
- Post duplicate or lightly spun comments across many communities.
- Target individuals for harassment, pressure, shame, or manipulation.
- Use hate speech or demeaning claims against protected classes.

## Fit Checklist

A candidate target is high-fit only when most of these are true:

- The thread explicitly discusses the same topic, argument, author, problem, or audience.
- The content being promoted answers a question, clarifies a disagreement, provides a source, or adds a useful perspective.
- The community rules permit self-promotion, source links, or creator participation.
- For an opinion-based or non-opinion specialist item, the draft would still be useful if the link were removed. For a general website introduction, the draft instead makes a brief, tailored contribution to a concretely relevant existing post, comment, question, thread, or debate before introducing the site.
- The user can disclose affiliation naturally.
- The thread is recent enough that a reply is not disruptive.

## Risk Flags

Mark a candidate as `do not post` or `needs manual rule check` when any of these appear:

- Rules ban self-promotion, outside links, creator content, or promotional replies.
- The community is support-oriented, grief-oriented, crisis-oriented, or otherwise vulnerable.
- The thread is a heated argument where a promotional link would escalate conflict.
- The same message would be sent to multiple places.
- The topic involves protected traits, medical decisions, legal advice, elections, minors, or crisis situations and the draft could mislead or manipulate.
- The user cannot disclose their connection to the content.

## Reply Standard

A good reply follows the promoted item's P&P standard:

1. Names the specific issue raised by the original poster.
2. Adds a reasoned argument for an opinion item, substantive specialist discussion for a non-opinion specialist item, or a brief contextual acknowledgment or extension for a general website.
3. Uses a transparent affiliation statement.
4. Invites disagreement or further discussion.
5. Avoids sales language, urgency, exaggeration, and repeated calls to action.

The general-website exception permits a small contribution because the promoted object is broad. It does not permit generic link dropping, unrelated replies, or lightly edited distribution of the same introduction across many targets.

Example structure:

```text
I think the key distinction in this thread is [specific point]. One way to frame it is [useful contribution]. I made/wrote [content] that walks through [specific relevance], including [specific detail]. Link: [URL]. No pressure, but it may be useful if you are comparing [topic].
```

## Manual Posting Checklist

Before an approved post is submitted:

- Confirm the account is allowed to participate in that community.
- Read the current rules, pinned posts, and recent moderator comments.
- Verify the exact draft is unique to that thread.
- Confirm the link is directly relevant and limited to one URL.
- Confirm affiliation is disclosed.
- Confirm the user approved the exact final message in Codex chat.
- Confirm no CAPTCHA, MFA, suspicious-login page, moderation warning, or rule warning is blocking the action. If one appears, stop and ask the user to handle it manually.
- Confirm the composer text matches the approved message before submitting.
